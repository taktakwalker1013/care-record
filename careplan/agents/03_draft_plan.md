# エージェント: 原案作成

## 役割
`needs.json` と型テンプレートを用い、第1表・第2表（必要なら第3表）の原案を組む。

## 入力
- `assessment.json`（意向・方針の素材）
- `needs.json`（ニーズ・根拠・優先度・型）
- 型テンプレート（`サービス組み合わせ` `役割分担` `第1表 総合的な援助の方針 記載例`）

## 出力
`careplan-draft.schema.json` 準拠のJSON。

## ルール
1. **第1表**
   - `intent_self` / `intent_family` は `assessment.voice`（主訴・意向）を根拠にする。素材にない意向を創作しない。
   - `needs_analysis`（課題分析の結果）は `assessment_summary` の overview・issues・strengths を根拠に、意向を踏まえた総括としてまとめる。
   - `policy`（総合的な援助の方針）は型テンプレートの記載例を"下敷き"にしつつ、この利用者の課題（褥瘡・医療依存・家族連携等）に具体化する。
   - 各セルは `schemas/cell_limits.json` の上限（実帳票の実測値ベース）に収める前提で書く。細かな圧縮は次段（字数調整）が担う。
2. **第2表**
   - `needs.json` の各ニーズに対し、長期／短期目標・サービス内容・種別／頻度を対応させる。`need_id` で紐づけを残す。
   - サービス種別・頻度は型テンプレートの「サービス組み合わせ（基本仕様）」「役割分担の原則」に整合させる。
   - 目標は測定可能・期間つき（例：長期6か月／短期3か月）にする。
3. **文字数**：この段では自然な長さでよい。上限内への圧縮は次段（字数調整）の仕事。
4. **創作禁止**：assessment / needs にない事実（サービス事業所名、頻度、疾患等）を足さない。不足は空欄のまま残し、後工程で人が補う。
