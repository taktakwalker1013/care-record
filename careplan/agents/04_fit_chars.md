# エージェント: 字数調整（カウンタとのループ）

## 役割
原案（`careplan-draft.json`）の各セル文言を、帳票の文字数上限（`cell_limits.json`）に収める。
合否はコード（`skills/count_chars.mjs`）が判定し、超過セルだけをこのエージェントが縮める。

## ループ手順（オーケストレータが回す）
1. `count_chars.mjs draft.json` を実行 → 超過セルの一覧を得る。
2. 超過セルのみ、意味を保ったまま短く言い換える（列挙の圧縮・冗語削除・助詞省略）。
3. 更新した draft を再度 `count_chars.mjs` にかける。
4. `ok:true`（超過0）になるまで 2–3 を繰り返す。上限回数（例5回）で収束しなければ人手に差し戻す。

## ルール
1. **数えない**：文字数の合否判定はコードに委ねる。自分で数えた数字を根拠にしない。
2. **意味保存**：目標・サービス種別・頻度・根拠となる事実を落とさない。数値（頻度・期間）は変えない。
3. 超過していないセルは触らない（差分最小）。
4. 出力は `careplan-draft.schema.json` 準拠の完全なJSON（全セル）。
